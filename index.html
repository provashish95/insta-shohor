How does javaScript work ?

JavaScript code/সোর্স ফাইল ব্রাউজারে execute করে ব্রাউজারগুলোর নিজস্ব javaScript engine দ্বারা । যেমন google chrome এ V8 engine/ Firefox এ Spidermonkey/ Edge এ Chakra/Safari used JavaScript Core । আমরা জানি JavaScript is an interpreted language তাই V8 engine বা V8 engine এর Internal JIT (just in time) compiler দ্রুত javaScript code execute করে দেয় । প্রথমে কোডগুলাকে লাইন টু লাইন চেক করে যেটাকে বলে Parser, যদি কোড চেক করে ভুল পায় তো সাথে সাথে আমাদের ব্রাউজারে ইনফর্ম করে দেয় আর যদি ভুল না পায় তাহলে নেক্সট স্টেপ Abstract Syntax Tree তৈরি করে। এবং পরবর্তীতে Interpreter ignition অন করে মেশিন কোড বা বাইটকোডে রুপান্তরিত হয় আর এই interpreter এর কাজটাকে দ্রুত করার জন্যই JIT ব্যবহার করা হয়। সাধারনত হাইলেভেল ল্যাঙ্গুয়েজ মেশিন বুঝতে পারে না তাই মেশিনকোডে কনভার্ট করতে হয় এবং কনভার্টেড কোড গুলোকে সিস্টেমে পাঠিয়ে দেওয়া হয় Execution এর জন্য, সর্বশেষ বাইটকোড বা মেশিনকোডটি সিস্টেম দ্বারা রান করার পর আমরা আমাদের আউটপুট দেখতে পাই । সাধারণত জাভাস্ক্রিপ্ট এভাবেই কাজ করে থাকে জাভাস্ক্রিপ্ট ইঞ্জিন দ্বারা।

What does javascript event loop do ?

আমরা জানি জাভাস্ক্রিপ্ট single threaded programming language, means that Where JavaScript code is run, runs in one line at a time manner and there is no possibility of running code in parallel। কিন্তু event loop এর মাধ্যমে asynchronous javaScript and multi threaded এর মত কাজগুলাকে ম্যানেজ করতে পারে । V8 engine এ Heap এবং Stack নামে দুইটা memory আছে । Heap হচ্ছে মেমোরি এরিয়া যেখানে ডাটা Randomly থাকে এবং Stack এ function গুলোকে স্টোর করে রেখে LIFO পদ্ধতিতে code execute করে থাকে। সাধারনত জাভাস্ক্রিপ্ট এ synchronously code execute হয় । যখন asynchronous কোনো function Call Stack এ যায় তখন Call Stack asynchronous function execution এর জন্য বসে না থেকে ঐ function টাকে popup করে Web API/ C++ API এ push করে দেয় এবং synchronous function গুলোকে execute করতে থাকে । Web API /C++ API তে asynchronous function এর কাজ শেষ হয়ে গেলে সেটা Event queue তে স্টোর হয়ে থাকে । আর এইখানেই Event loop এর কাজ শুরু হয়, কারন event loop প্রথমে Call Stack চেক করে দেখে main() function বাদে বাকি function এর কাজ শেষ হইছে কিনা, যদি শেষ হয়ে থাকে তাহলে priority অনুযায়ী Event Queue থেকে FIFO পদ্ধতিতে Event loop asynchronous function গুলোকে সিরিয়ালি Call Stack এ push করে দেয় এবং Call Stack asynchronous function গুলা Execute করে। সুতরাং আমরা বলতে পারি Event Loop এর কাজই হচ্ছে Stack memory and Event Queue কে মনিটর করা এবং Rules অনুযায়ী Queue থেকে function গুলাকে Stack memory তে পৌছিয়ে দেওয়া Execution এর জন্য ।

Difference between local storage and session storage

Local Storage and Session Storage is a part of Web Storage API,যা আমাদেরকে ব্রাউজারে ডাটা store,update,read and delete করতে সাহায্য করে । Local and Session Storage প্রায় একই তবে কিছু কিছু পার্থক্য আছে যা নিচে তুলে ধরা হলঃ

Local Storage এ ডাটা পার্মানেন্টলি স্টোর করা যায় । কিন্তু Session Storage এ ডাটা শুধুমাত্র ঐ একটা session এর জন্য স্টোর করা যায়, পার্মানেন্টলি ডাটা স্টোর করা যায় না । কারন Session Storage একটা ফিক্সড ট্যাব বা window এর জন্য কাজ করে । যখন ব্রাউজার/ট্যাব/window close করার সাথে সাথে Session Storage এর ডাটা auto ক্লিয়ার হয়ে যায় কিন্তু Local Storage এর ডাটা auto ক্লিয়ার হবে না । কারন ব্রাউজার বা ট্যাব close করে আবার open করলেও Local Storage এ একই ডাটা দেখা যাবে। তবে Local Storage এর ডাটা via javascript or user manually clear করতে পারে । অন্যদিকে Local Storage এর সাইজ ৫/১০ এমবি এবং Session Storage এর সাইজ ৫ এমবি ।
